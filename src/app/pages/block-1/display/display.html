1.4. Display и Поток документа

Свойство display определяет тип рендеринга элемента в нормальном потоке документа (Normal Flow) [1] и характер его взаимодействия со смежными блоками.

Интерактивная песочница

Переключай режимы отображения для Второго элемента (Target) и наблюдай, как ведут себя соседние блоки в потоке.

Блочный (block): Элемент занимает всю ширину контейнера и всегда переносится на новую строку. К нему можно применять любые размеры и отступы.

} @case ('inline') {

Строчный (inline): Элемент встраивается в строку текста. Ему нельзя задать ширину и высоту, а вертикальные padding/margin не раздвигают окружающий поток.

} @case ('inline-block') {

Строчно-блочный (inline-block): Элемент располагается в строке, но ведет себя как блок: принимает свойства ширины, высоты и все виды отступов.

} @case ('none') {

Убран из потока (none): Элемент полностью удаляется из макета, а его соседи схлопываются, моментально занимая освободившуюся площадь.

} } @if (displayMode() !== 'none' && visibilityMode() === 'hidden') {

Обратите внимание: При visibility: hidden элемент скрывается визуально, но его физическое место в потоке сохраняется.

Сравнительный анализ и правила потока

Попытка задать ширину (width), высоту (height) или вертикальные внешние отступы (margin-top / margin-bottom) для строчных элементов вроде <span>, <a> или <strong> ни к чему не приведет. Чтобы размеры применились, измените их режим на inline-block или block.

Это классический вопрос на технических собеседованиях:

  • display: none — элемент полностью вырезается из дерева рендеринга. Браузер не выделяет под него место, соседние элементы сдвигаются (схлопывание потока).
  • visibility: hidden — элемент становится полностью невидимым (как прозрачный), но его геометрические размеры и место в потоке остаются нетронутыми. Элементы вокруг не сдвигаются.
