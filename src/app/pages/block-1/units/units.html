1.5. Единицы измерения в CSS

Выбор правильной системы измерения — основа профессиональной адаптивной верстки. CSS предлагает богатый арсенал величин, делящихся на абсолютные и относительные, каждая из которых решает свои специфичные задачи.

1. Шрифт-зависимые единицы (Font-Relative)

Эти единицы привязаны к размерам символов активного шрифта. Они незаменимы для построения пропорционального интерфейса, масштабирующегося вместе с текстом.

  • rem (Root EM) — рассчитывается строго от размера шрифта корневого элемента <html>. По умолчанию в подавляющем большинстве браузеров 1rem = 16px. Рекомендуется использовать для отступов, ширины колонок и размеров всех шрифтов на сайте, чтобы сохранить доступность интерфейса (пользователи со слабым зрением могут увеличить базовый шрифт в браузере, и вся верстка пропорционально вырастет).
  • em — имеет двойственную природу. Если используется в свойстве font-size, то вычисляется от шрифта родительского контейнера. Если в любых других свойствах (width, padding, margin) — от собственного шрифта этого же элемента.
  • ch (Character) — равна ширине глифа цифры «0» (ноль) текущего шрифта. Это лучшая единица для ограничения ширины текстовых блоков.

Офтальмологи и специалисты по UX доказали, что человеческий глаз лучше всего воспринимает текст, если в одной строке находится от 45 до 75 символов. Задавая текстовым блокам свойство max-width: 60ch;, вы гарантируете, что строки будут иметь эталонную читаемость на любых мониторах и шрифтах.

2. Единицы вьюпорта (Viewport-percentage)

Единицы вьюпорта рассчитываются от габаритов видимой области окна браузера (viewport). 1% вьюпорта равен 1 единице.

  • vw (Viewport Width) — 1% от ширины окна браузера. Например, при ширине экрана 1024px, 10vw составят 102.4px.
  • vh (Viewport Height) — 1% от высоты окна браузера. Идеально подходит для создания полноэкранных секций (Hero block: height: 100vh;).
  • vmin — выбирает наименьшее значение между vw и vh. Удобно для адаптации квадратных элементов (например, модальных окон или игровых полей), чтобы они гарантированно влезали в экран как на смартфонах, так и на десктопах.
  • vmax — выбирает наибольшее значение между vw и vh.

3. Мобильная триада высот (dvh, svh, lvh)

Классическая единица 100vh имеет огромную проблему на мобильных устройствах. Мобильные браузеры (iOS Safari, Chrome) содержат динамически скрывающиеся панели интерфейса (адресную строку и меню навигации).

По спецификации 100vh рассчитывается по размеру экрана при скрытых панелях (Large Viewport). Из-за этого, когда пользователь только открывает сайт, нижняя часть элемента height: 100vh оказывается перекрытой и уходит под навигационную панель телефона. Для решения этой боли были введены новые динамические единицы:

  • svh (Small Viewport Height) — высота вьюпорта при полностью открытых панелях интерфейса. Это гарантированно видимая область экрана без какого-либо перекрытия.
  • lvh (Large Viewport Height) — высота вьюпорта при полностью скрытых панелях браузера (аналог классического vh в мобильных браузерах).
  • dvh (Dynamic Viewport Height) — динамическая высота. Она непрерывно пересчитывается браузером в реальном времени при скролле пользователя (когда адресная строка плавно сворачивается или разворачивается).

Так как при использовании dvh браузеру приходится пересчитывать геометрию элементов при каждом пикселе скролла (когда скрывается адресная строка), злоупотребление этой единицей на множестве анимированных элементов может вызвать микрофризы рендеринга на старых мобильных процессорах. Для большинства типовых задач (фиксированное боковое меню, шторки) лучше подходит стабильный 100svh.

Хотя поддержка современных единиц высоты отличная (Safari 15.4+, Chrome 108+, Firefox 101+), для старых устройств всегда пишите двухстрочный фоллбек, используя свойство каскада (браузер проигнорирует то, что не понимает):

.hero-section {{ '{' }}
  height: 100vh;  /* Для старых браузеров */
  height: 100dvh; /* Для современных мобильных устройств */
{{ '}' }}
